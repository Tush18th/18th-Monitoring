# Backend Core APIs (`apps/api`)

The primary source validating cross-tenant integrations mapping security protocols safely across streams enforcing dashboard persistence efficiently.

## 🎯 Architecture Focus

Built explicitly using `Fastify` for low-overhead routing parsing high-velocity JSON requests safely. It bridges complex `dashboard.service` models efficiently validating states gracefully. 

## 🌐 Endpoints Available
- `POST /api/v1/auth/login`: Handles standard password decryption structures checking mock bounds emitting Token arrays safely.
- `GET /api/v1/projects`: Project validation lists parsing `super_admin` bounds gracefully.
- `GET /api/v1/dashboard/summaries`: Central KPI parser returning metrics based upon specific DTO scopes:
    - Filters: `{ siteId, timeRange }`.
    - Computes values structurally iterating in-memory records locally securely mapping statuses to `{healthy, warning, critical}`.
- `GET /api/v1/dashboard/alerts`: Dynamic filtering tracking architecture breaches returning limits.
- `POST /api/v1/simulate`: Trigger route bridging standard `TopBar` ingestion clicks natively deploying Mock schemas evaluating limits automatically.

## 🔐 Environment Needs

The API evaluates `.env` properties protecting the JWT hash boundary securely mapping intercepts effectively:
- `PORT` (Usually 4000)
- `JWT_SECRET` (Core Hash constraint required natively bypassing legacy `hardcoded_demo_salt` boundaries globally).
